**Sandboxer 0.2.1**

Sandboxer is a Crystal shard that runs shell commands inside a platform-native sandbox with a configurable access policy. It works on macOS (via `sandbox-exec` / Seatbelt) and Linux (via `bwrap` / Bubblewrap), presenting both as a single clean API.

**Changes**

- **Expanded macOS BASELINE** — added allowances for Darwin/CoreFoundation plumbing, syslog (as a local socket, not gated by `allow_network`), and DNS resolver config (`/etc/resolv.conf`, `/etc/hosts`). These are needed by nearly any real-world process and previously had to be worked around in caller policies.
- **Symlink resolution in profiles** — policy paths are now resolved via `realpath` before being written to the SBPL profile. SBPL matches against the kernel-resolved path, so unresolved symlinks would silently match nothing. Falls back to `expand_path` for paths that don't exist yet.
- **Spec reorganisation** — specs now mirror the source tree under `spec/sandboxer/`, matching the existing `spec/presets/` convention.
